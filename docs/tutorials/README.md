# Tutorials

Tutorials sind Lernpfade.

Ziel: Jemand mit wenig Projektkontext soll Schritt für Schritt zu einem funktionierenden Ergebnis kommen, ohne vorher die gesamte Architektur verstanden zu haben.

## Geplante Tutorials

1. **Erstes Video lokal erzeugen**
   - [`first-local-pipeline-run.md`](first-local-pipeline-run.md)
   - Setup
   - Testdaten
   - Pipeline-Run ohne Upload

2. **Dashboard lokal starten und bedienen**
   - Service starten
   - Oberfläche verstehen
   - ersten Job nachvollziehen

3. **Vom Song zum Publish-Artefakt**
   - Audio
   - Thumbnail
   - Metadaten
   - Render
   - optional Upload

## Schreibregeln

Ein Tutorial soll:
- ein klares Anfangssetup haben
- konkrete Schritte enthalten
- ein überprüfbares Endergebnis liefern
- keine Architektur-Abhandlung sein

Wenn Erklärung nötig ist, lieber auf `docs/explanation/` oder andere passende Stellen verlinken.

- [`dashboard-local-start.md`](dashboard-local-start.md)
- [`song-to-output.md`](song-to-output.md)
