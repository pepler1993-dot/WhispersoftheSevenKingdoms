# Agent Operations Manual

Diese Sektion ist für **arbeitende Agenten im Projekt**.

Nicht für Endnutzer, nicht für allgemeine Contributor-Einführung, sondern für Bots/Agenten, die im Repo tatsächlich Änderungen machen, dokumentieren, reviewen, synchronisieren und pushen.

## Zweck

Ein Agent soll hier schnell Antworten finden auf:
- Wie arbeite ich sicher in diesem Repo?
- Welche Regeln gelten für Änderungen?
- Wie pushe ich sauber nach GitHub?
- Wie benutze ich den Sync-Service?
- Wie vermeide ich Kollisionen mit anderen Agenten?

## Inhalte

- [`WORKING_IN_THIS_PROJECT.md`](WORKING_IN_THIS_PROJECT.md) — Regeln, Arbeitsweise, Branching, Push-Verhalten
- [`GITHUB_AND_PAT.md`](GITHUB_AND_PAT.md) — GitHub-Push mit PAT, Sicherheitsregeln, typische Git-Kommandos
- [`SYNC_SERVICE.md`](SYNC_SERVICE.md) — Nutzung des Agent-Sync-Service im Projektkontext

## Wichtige Abgrenzung

Diese Doku beschreibt **Agenten-Operations**.
Sie ersetzt nicht:
- `PROJECT_STATUS.md` für die operative Wahrheit
- `ROADMAP.md` für Prioritäten
- normale Guides/Tutorials für Menschen

## Reihenfolge für neue Agenten

1. `README.md` im Repo
2. `PROJECT_STATUS.md`
3. diese Agent-Doku
4. erst dann Änderungen machen
