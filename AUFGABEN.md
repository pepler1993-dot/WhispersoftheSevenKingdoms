# Aufgaben – Whispers of the Seven Kingdoms

Ziel: Videos und Musik zum Einschlafen im Game of Thrones-Stil posten und das Projekt gemeinsam weiterentwickeln.

## Technische Arbeitspakete

### 1. Fundament & Struktur
- [ ] Verzeichnisstruktur für Input, Work, Output, Templates und Scripts anlegen
- [ ] Benennungskonventionen für Songs, Assets und Exporte definieren
- [ ] Standard für Projekt-Metadaten festlegen (`song.json` oder `song.yaml`)
- [ ] Pflichtdateien pro Song definieren
- [ ] Rollen und Zuständigkeiten im Team dokumentieren

### 2. Song-Planung
- [ ] Vorlage für Song-Briefing erstellen
- [ ] Themenliste für Häuser, Orte, Figuren und Stimmungen aufbauen
- [ ] Zielplattformen pro Song festlegen
- [ ] Dauer- und Formatvorgaben definieren

### 3. Audio-Produktion
- [ ] Standard-Eingangsformate für Rohmaterial festlegen
- [ ] Workflow für Komposition / Generierung / Bearbeitung dokumentieren
- [ ] Exportprofile für Master und Plattformversionen definieren
- [ ] Audio-QA-Kriterien dokumentieren (Lautheit, Peaks, Länge, Fehler)
- [ ] Technische Audio-Prüfung automatisierbar machen

### 4. Visuals & Video
- [ ] Stil für Cover-Art und YouTube-Visuals festlegen
- [ ] Thumbnail-Spezifikation definieren
- [ ] Video-Template für lange Uploads definieren
- [ ] ffmpeg- oder Script-basierte Render-Pipeline planen
- [ ] Optional: Shorts-/Teaser-Format vorbereiten

### 5. Metadaten & Content-Paket
- [ ] Titelregeln definieren
- [ ] Beschreibungsvorlagen erstellen
- [ ] Tag-System / Keyword-Sets aufbauen
- [ ] Playlist-Zuordnungen dokumentieren
- [ ] Plattformfelder für YouTube, Spotify, SoundCloud festlegen

### 6. QA & Review
- [ ] Preflight-Checkliste definieren
- [ ] Fehler- und Warnsystem dokumentieren
- [ ] Review-Prozess vor Veröffentlichung festlegen
- [ ] Freigabeformat für veröffentlichungsfertige Songs definieren

### 7. Publishing
- [ ] YouTube als erste Zielplattform konkret ausarbeiten
- [ ] Upload-Prozess für YouTube dokumentieren
- [ ] Authentifizierungsstrategie sicher festlegen
- [ ] Upload-Logs und Fehlerbehandlung planen
- [ ] Danach Spotify/SoundCloud auf Machbarkeit prüfen

### 8. Betrieb & Zusammenarbeit
- [ ] Branch-Workflow konsequent einhalten
- [ ] PR-/Review-Routine etablieren
- [ ] README und Agent-Dokumentation aktuell halten
- [ ] Secrets niemals im Repo ablegen

## Priorisierte Umsetzungsreihenfolge

### Phase 1 – Basis schaffen
- [ ] Ordnerstruktur anlegen
- [ ] Metadaten-Schema definieren
- [ ] Naming-Regeln dokumentieren
- [ ] YouTube als MVP-Plattform festlegen

### Phase 2 – Produktionsstrecke bauen
- [ ] Audio-QA standardisieren
- [ ] Thumbnail-/Video-Template definieren
- [ ] ersten Video-Render-Prototyp bauen
- [ ] Beschreibung/Titel aus Metadaten generieren

### Phase 3 – Veröffentlichung absichern
- [ ] Upload-Paketstruktur festlegen
- [ ] YouTube-Upload reproduzierbar machen
- [ ] Logging und Fehlerbehandlung ergänzen
- [ ] Review vor Veröffentlichung verbindlich machen

### Phase 4 – Ausbau
- [ ] Multi-Plattform-Uploads prüfen
- [ ] Social-Media-Teaser ergänzen
- [ ] Feedbackschleife und Optimierung einbauen
