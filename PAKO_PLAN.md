# Pako Plan – Ausbau zum Content-System

## Ziel
Aus einem einzelnen Song-Workflow eine stabile, skalierbare Content-Pipeline machen.

## 1. Kurzfristig: aktuellen Workflow stabilisieren
- Colab nicht als verlässliche Dauerlösung betrachten
- `prepare/resume`-Ablauf sauber halten
- Upload-Pipeline robust machen
- Fehlerfälle klar behandeln: Abbruch, fehlendes Audio, Rate Limits, Upload-Fehler

## 2. Aus einem Song mehrere Assets machen
Jeder Song soll nicht nur als Full Release enden, sondern als Paket:
- Full Song
- YouTube Video
- YouTube Short
- TikTok/ Reel Clip
- Thumbnail-Varianten
- Metadaten/Textvarianten

Ziel: 1 Input → mehrere Outputs

## 3. Formate statt Zufall
3 wiederkehrende Reihen aufbauen:
- episch / düster
- meme / shitpost
- lore / charakterbasiert

So wird aus Einzelcontent eine Marke.

## 4. Plattformen erweitern
Nach YouTube systematisch prüfen:
- Shorts
- TikTok
- Instagram Reels
- SoundCloud
- später Distribution auf Spotify/Apple Music

Nicht einfach blind überall hochladen, sondern formatgerecht.

## 5. Automatisierung ausbauen
Automatisieren, was repetitiv ist:
- Prompt → Job
- Song → Video
- Titel/Beschreibung/Hashtags
- Varianten für Clips
- Upload-Vorbereitung
- einfache Ergebnis-Dokumentation

## 6. Messbar arbeiten
Für jede Veröffentlichung festhalten:
- was wurde gepostet
- auf welcher Plattform
- welche Variante
- was performt besser

Nicht nur machen. Lernen.

## 7. Empfehlung
Erst System und Zuverlässigkeit, dann Skalierung.
Ein chaotischer Massen-Output bringt nur chaotischere Probleme.
