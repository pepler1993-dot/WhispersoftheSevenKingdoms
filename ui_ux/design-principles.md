# Design Principles

## Product stance
Design for a modern SaaS platform for automated multi-platform content production and publishing.

## Core principles
1. Output first
   - show what was published and what is scheduled next

2. Problems visible
   - urgent issues and blockers must be obvious

3. Trust through system status
   - automation health, channel health, and approval state must be easy to understand

4. Strong hierarchy
   - the most important information must be visible in the first 5-10 seconds

5. High scanability
   - users should understand the page by scanning, not by reading everything

6. Actionability
   - important next actions must be obvious

7. Thumbnail-first where useful
   - published content should often show thumbnail previews

8. Consistent states
   - published, scheduled, processing, blocked, failed, review-needed must be clearly distinct

9. Generic SaaS, not theme-locked
   - the UI must work across many brands, customers, channels, and content types

10. Reusable patterns
   - consistent cards, badges, filters, tables, progress bars, and section layouts

## Preferred overview order
1. Global controls / primary actions
2. KPI row
3. Recently published
4. Needs attention
5. Upcoming schedule
6. Channel health
7. Pipeline / approval health
8. Active productions

## Avoid
- buried errors
- overly decorative layout
- weak CTA visibility
- long dense lists without hierarchy
- too many equally weighted sections
- technical details with low user value
